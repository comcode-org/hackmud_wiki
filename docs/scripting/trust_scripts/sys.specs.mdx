---
title: sys.specs
---

sys.specs returns the user's current specifications.

## Syntax

### CLI

```
sys.specs
```

### Script

```
#ns.sys.specs()
```

### Parameters

No known parameters.

### Return

Returns a string.

#### CLI

```
>>sys.specs
    <ASCII art representation of user-class>

<username> (<user-class>)

tier: <number out of 4>

hardline_count: <number of remaining hardlines>
next_hardline: <seconds until next hardline refresh>

architect(<score>) scavenger(<score>) executive(<score>) infiltrator(<score>) junkrack(<score>)

channel_count: <number of channels user can be in>

gc_max: 9Q7T199B254M740K991GC

upgrade space
slots: <current upgrades held>/<max upgrade space>
loaded: <current upgrades loaded>/<max load space>

script space
publics: <number of public scripts a user can #up>
slots: <total number of scripts user can #up>
chars: <current character limit>
```

#### Script

Same as CLI.

## Example

```js
function(context, args)
{
	return #hs.sys.specs()
}
```
